# Mission

## Project
Puppeteer OS v1 Scaffold

## Mission
Build a local-first, repo-backed project operating system that lets one human run disciplined AI-assisted project workflows without direct model APIs.

## Desired outcome
A reusable project scaffold that can ingest source material, assign structured work to subscription chatbots, write outputs back into the repo, and surface current state in a dashboard.

## Constraints
- No direct model API access
- Human-in-the-loop execution
- Repo as canonical truth
- Local-first operation
- File-backed dashboard state
- Manual fallback for all critical loops

## Non-negotiables
- No unsupported claims
- No hidden state outside the repo
- No output without a write-back path
- No approval by implication
- No browser automation in the critical path for v1

## Current phase
Bootstrap

## Definition of done
The scaffold is usable when one real task can move from inbox -> active -> review -> done using worker packs, saved outputs, explicit approval, and refreshed dashboard feeds.
