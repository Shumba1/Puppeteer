# T-003 — QA the first project ingestion flow

## Objective
Review whether the first project documentation is being routed into the correct repo folders and surfaced properly in the dashboard.

## Why it matters
A beautiful command centre is useless if the ingestion flow creates state drift.

## Inputs
- `MISSION.md`
- `STATE.json`
- `skills/qa.md`
- `docs/specs/dashboard-module-spec.md`
- `docs/specs/observability-layer-spec.md`

## Worker role
Critic

## Output path
`outputs/drafts/ingestion-qa.md`

## Output format
Markdown with headings:
- What is working
- Where state is drifting
- Dashboard gaps
- Observability gaps
- Fixes to apply now

## Acceptance criteria
- folder placement is sensible
- dashboard visibility is adequate
- any drift risks are explicit
- observability gaps are concrete
