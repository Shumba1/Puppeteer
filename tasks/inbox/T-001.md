# T-001 — Bootstrap plan for the first manual loop

## Objective
Create a practical bootstrap plan that proves one full manual operator cycle using this scaffold.

## Why it matters
This is the first evidence that the system works as an operating substrate rather than just a folder structure.

## Inputs
- `MISSION.md`
- `STATE.json`
- `skills/planning.md`
- `skills/risk-review.md`

## Worker role
Driver

## Output path
`outputs/drafts/bootstrap-plan.md`

## Output format
Markdown with headings:
- Executive Summary
- Proposed Loop
- Risks
- Approval Gate
- Next Actions

## Acceptance criteria
- clear sequence of actions
- explicit approval point
- explicit fallback if a worker response is weak
- no unsupported assumptions
