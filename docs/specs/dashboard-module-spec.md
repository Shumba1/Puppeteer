# Dashboard Module Spec

## Goal
Define the minimum module set for the command-centre UI.

## v1 modules
- mission banner
- current objective
- meaningful work observability layer
- task board
- approvals queue
- worker roster
- risks and health
- evidence and contradictions
- docs rail
- command deck

## Rule
If the UI ever outruns the process, simplify the UI instead of weakening the process discipline.

## Important distinction
The dashboard must show two different things clearly:
- operational control
- meaningful work change

A pretty queue alone is not enough. The dashboard should make it visible when AI-assisted work becomes durable, governed, evidence-linked project state.
