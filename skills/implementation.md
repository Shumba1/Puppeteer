# Skill: Implementation

## Description
Use when a plan must be translated into code, structure, documents, or other concrete project artifacts.

## Required inputs
- task file
- relevant plan or synthesis file
- output path
- acceptance criteria

## Process
1. Confirm the deliverable and output path.
2. Confirm the minimum viable implementation.
3. Produce the artifact cleanly.
4. Note assumptions made.
5. Flag anything that still requires review.

## Output rules
- Follow the requested structure exactly
- Keep changes scoped to the task
- Do not broaden the work without approval
