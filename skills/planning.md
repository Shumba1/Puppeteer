# Skill: Planning

## Description
Use when a mission or task needs to be broken into phases, dependencies, approval points, and immediate next actions.

## Required inputs
- `MISSION.md`
- `STATE.json`
- relevant task file
- any directly relevant evidence or source notes

## Process
1. Read mission and current state.
2. Define the concrete objective in one sentence.
3. Break the work into phases or steps.
4. Identify dependencies and blockers.
5. Identify risks and missing information.
6. Propose explicit approval points.
7. Recommend the next 3-5 actions only.

## Output format
```md
# Planning Output

## Objective
...

## Proposed phases
1. ...
2. ...

## Dependencies
- ...

## Risks
- ...

## Approval points
- ...

## Recommended next actions
- ...
```

## Escalate when
- the mission is unclear
- constraints conflict
- the output path is missing
